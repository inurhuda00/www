After the operator sends payout requests, the provider will process it. Once the provider gets the transaction status (success or failed), the provider will call this API to forward the transaction status.

```bash cURL
curl --request POST \
  --url https://s88pay.net/api/v1/transaction/resend-callback/SKU20210909025705 \
  --header 'Content-Type: application/json' \
  --data '{
      "key": "<string>",
      "transaction_code": "<string>",
      "transaction_no": "<string>"
    }'
```

```json Response after decrypting
{
  "transaction_no": "",
  "transaction_code": "",
  "transaction_status": "",
  "transaction_amount": "",
  "transaction_fee": "",
  "currency_code": "",
  "transaction_ref": ""
}
```

Provider will send the parameter on key format, then operator need to decrypt it using [encrypt/decrypt](/api-reference/encrypt-decrypt), you need to decrypt with `api_key` and `api_secret`.

Besides the key, we also send the transaction code in the `transaction_code` parameter and the transaction number in the `transaction_no` parameter that is not encrypted.
